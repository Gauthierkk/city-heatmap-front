# Grocery Heatmap

Interactive map of switchable business categories — grocery stores, specialty
food shops, and fitness venues — across Paris, NYC, and Austin, with a
proximity heatmap: enter your address and see which places are closest,
filtered by type. Paris additionally offers a **Transit** category (metro, RER,
tram, train, VAL and major stations — closest-station search just like shops)
and a **Trees** category: an unlabelled street-tree density heatmap (clipped to
the city outline) with an adjustable spread (10–50 m).

Built with React, Vite, TypeScript and MapLibre GL JS. Store data ©
[OpenStreetMap](https://www.openstreetmap.org/copyright) contributors (ODbL),
fetched via the Overpass API; geocoding by Nominatim.

## Getting started

```bash
npm install
npm run dev        # http://localhost:5173
```

## Store data

The bundled GeoJSON lives at the repo root under `data/places/<city>/` (the
`food` and `fitness` store files, plus `transit` and a `trees` Point
FeatureCollection — each tree tagged with its species — for Paris) and
`data/boundaries/<city>.geojson` (overlay clip
boundary), served
same-origin at runtime (a small Vite plugin maps `./data/*` → `/data/*`). It is
**generated by a separate worker repo**,
[`city-heatmap-data`](../city-heatmap-data), which runs weekly on a local
server, regenerates the data into this repo's `data/`, and commits + pushes so
GitHub Pages redeploys. This repo carries no Python — see the worker repo's
README for the fetch commands and refresh runbook.

Only **Paris** is on the weekly refresh. The **NYC** and **Austin** datasets are
one-off snapshots that are not updated regularly, so the app shows a
data-freshness disclaimer when either is selected.
